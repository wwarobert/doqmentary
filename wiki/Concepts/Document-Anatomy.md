# Document Anatomy

A doqmentary **document** is a folder under `documents/` that holds one markdown file
per configured section, plus an assembled wiki output.

## Folder structure

```
documents/
└── my-solution/
    ├── doqmentary.yaml      ← optional per-document config override
    ├── sections/
    │   ├── summary.md
    │   ├── project-background.md
    │   ├── detailed-scope.md
    │   └── ...              ← one file per configured section
    └── wiki/                ← assembled output (git-ignored, regenerable)
        ├── index.md         ← home page (synthesized from Summary)
        ├── project-background.md
        ├── detailed-scope.md
        ├── ...
        └── _sidebar.md
```

## Section files

Each section file is a markdown file with YAML front matter:

```markdown
---
id: project-background
title: Project background
status: draft          # draft | ingested | authored | synthesized
layer: situation       # situation | enterprise | synthesis
---

The body content goes here.
```

The `status` field tracks how the section was populated:

| Status | Meaning |
|--------|---------|
| `draft` | Scaffolded, not yet filled |
| `ingested` | Populated from deck text via `doqmentary ingest` |
| `authored` | Filled by the author skill in Copilot |
| `synthesized` | Generated by the author skill from other sections (derived only) |

## Section types

Sections fall into three types determined by the `layer` field in the config.
See [Reference/Section-Types](Reference/Section-Types) for the full breakdown.

| Layer | Filled by | Example |
|-------|-----------|---------|
| `situation` | Author in Copilot (interviewed per solution) | Project background, Scope |
| `enterprise` | Author skill from enterprise library | Key design decisions, Principles |
| `synthesis` | Author skill from other sections (derived) | Summary |

## The assembled wiki

Running `doqmentary assemble` produces a linked wiki under `documents/<solution>/wiki/`:

- One page per **body section** (non-derived)
- A **home page** (`index.md`) synthesised from the Summary section, with links to detail pages
- A **navigation manifest** (`_sidebar.md`) for the configured output target

The wiki directory is **git-ignored** — it is always regenerable from the section
sources via `doqmentary assemble`.

## Configuration layering

Section definitions and output settings come from `doqmentary.yaml`. A
per-document override at `documents/<solution>/doqmentary.yaml` can replace any
value. Per-document values win; omitted values inherit from the global config.

See [Reference/Configuration-Schema](Reference/Configuration-Schema) for the full schema.
